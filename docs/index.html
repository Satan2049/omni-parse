How it works

A simple pipeline from input to RAG-ready artifacts.

01
Paste a URL or HTML Use the Next.js workspace — configure timeouts, user-agent, and JS rendering.
02
Fetch & extract reqwest or headless Chrome/Edge fetches the page; the readability engine extracts title, body, metadata, and images.
03
Preview & export Inspect Markdown, JSON, or text in the preview panel — download or convert via the API.

HTTP API

Integrate extraction into scripts and automation. The API runs on http://127.0.0.1:8000 in dev or embedded inside the desktop app.

Method Path Description
POST /extract Extract title, markdown, metadata, and image URLs
POST /convert Convert text or markdown to PDF, TXT, or MD download
GET /images/download Download a resolved public image URL through the API proxy
GET /settings Read server arrangement values
PUT /settings Update arrangements (persisted to .env)
GET /health Health check for orchestration and the UI status badge 
# Example: extract with full-size image resolution
curl -X POST http://localhost:8000/extract \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com/gallery", "resolve_fullsize_images": true}'

Quick start (Windows)

Prerequisites: Rust, Node.js 20+, and Google Chrome or Microsoft Edge (for JS rendering). Double-click start.bat in the repo root. 

# Services after launch
UI       →  http://localhost:3000
API      →  http://localhost:8000

# Desktop installer (release)
.\scripts\build-desktop.ps1

Prefer manual setup? See the README for API, frontend, and Tauri build steps.
